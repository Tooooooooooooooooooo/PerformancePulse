# PerformancePulse · 绩效图表系统

基于 Flask + Chart.js 的团队绩效数据可视化平台，支持 Excel 上传解析、多维度图表分析、KPI 统计与自定义配置。

## 快速启动

```bash
pip install flask pandas openpyxl xlrd
python app.py
```

- 前台：http://127.0.0.1:5000/performance
- 后台：http://127.0.0.1:5000/admin（默认账号 `admin` / `admin123`）

数据库自动初始化于 `data/performance_pulse.db`。

## 数据库结构

| 表名 | 用途 |
|------|------|
| `users` | 管理员账号（用户名、密码哈希、角色） |
| `people` | 人员名册（姓名、是否在前台展示） |
| `performance_records` | 工作日志记录（人员、甲方、任务描述、数量、单位、日期） |

| `app_config` | 应用配置（Chart.js URL、主题、标题、小数位等，扩展字段存于 `extra_json`） |

## 前台功能（/performance）

### 筛选条件

- 按**人员**（下拉选择，来自后台可显示人员列表）筛选
- 按**开始/结束日期**筛选
- 支持组合筛选，点击「应用筛选」生效
- 点击「重置」清空所有条件
- **筛选状态持久化**：使用 `sessionStorage` 记录当前人员、日期、激活筛选项和对比人员，刷新页面后自动恢复

### 顶部统计卡片

| 卡片 | 说明 |
|------|------|
| 任务总量（KPI） | 当前筛选范围内所有自定义筛选项 KPI 值之和 |
| 当月预测 KPI | 根据当月已过天数与已完成 KPI 线性外推的月末预测值（橙色显示），仅当月数据可见时显示 |
| 涉及人员 | 筛选结果中出现的不同人员数 |
| 涉及天数 | 筛选结果中出现的不同日期数 |
| 员工未工作天数 | 以数据最早至最晚日期为基准（含周六日），展示每位员工未提交工作日志的天数，带进度条和占比 |

### 模块：KPI 总量趋势

- 独立卡片，位于自定义筛选项模块上方（顺序可在后台调整）
- 折线图，每条线代表一个自定义筛选项，展示各筛选项 KPI 随日期的变化趋势
- 每个数据点上方显示数值标签
- 「显示/隐藏总 KPI 走势」按钮：开启时叠加一条白色虚线显示所有筛选项 KPI 之和，同时隐藏分项曲线；关闭时恢复各分项

### 模块：自定义筛选项 KPI 总览

- 以卡片网格展示每个启用的自定义筛选项的**汇总数据**（总量、KPI 值、系数）
- **点击任意筛选项高亮选中**（金色边框+背景），并联动更新下方四张详情图
- 四张详情图：
  - 原始数量趋势（折线）
  - 甲方占比（原始数量，条形图）
  - KPI 值趋势（折线）
  - 甲方占比（KPI 值，条形图）

### 模块：人员 KPI 对比

- **不受筛选条件影响**，始终基于全量数据
- 展示所有人员标签，点击加入/移除对比（最多同时对比任意多人）
- 选中人员以彩色圆角标签显示（与图表颜色一致），未选中显示灰色
- 对比折线图：每人一条曲线，按日期展示 KPI 值，带数据标签
- 「清空对比」按钮一键重置

### 模块：每日/月度/人员图表

- 三张柱状图，分别展示：按日期聚合的 KPI 总量、按月份聚合的 KPI 总量、按人员聚合的 KPI 总量
- 每个柱子顶部显示数值标签

### 模块：明细表

- 展示当前筛选范围内所有工作日志记录（日期、人员、甲方、任务描述、数量）
- 支持滚动（最大高度 360px），**不显示滚动条**
- 与上方模块保持间距

### 模块排列顺序

共 5 个可排序模块：KPI 总量趋势、自定义筛选项 KPI 总览、人员 KPI 对比、每日/月度/人员图表、明细表。顺序由后台「前端模块顺序」拖拽配置后保存生效。

### 配色方案切换

- 右上角提供 5 个彩色圆点，点击即时切换整站配色
- 切换结果保存至 `localStorage`，下次访问自动恢复
- 若访客从未手动选择，则使用后台设置的默认配色方案

| 方案 | 主色 |
|------|------|
| 暗金（默认） | 金色 `#c8a97e` |
| 深海蓝 | 蓝色 `#60a5fa` |
| 深林绿 | 绿色 `#4ade80` |
| 玫瑰红 | 粉红 `#fb7185` |
| 明亮 | 紫色 `#7c5cbf`（浅色背景） |

### 导出

- 「导出 Excel」按钮：将当前筛选范围内的明细记录导出为 `.xlsx` 文件

## 后台功能（/admin）

后台需要登录才能访问（默认 `admin` / `admin123`）。

### 数据上传

- 支持上传 `.xls` / `.xlsx` 格式的工作日志 Excel 文件
- **增量合并**：只替换本次文件日期范围内的旧记录，其他历史数据保留
- 人员表只增不减，历史人员不丢失
- 上传成功后显示合并记录数和日期范围
- **格式检查**：上传后自动检测单元格书写格式问题，在上传卡片内逐条列出（行号、人员、日期、原文、问题描述），数据正常写入不受影响

#### 格式检查项目

| 问题类型 | 示例 |
|----------|------|
| 逗号分隔多任务（应用换行或分号） | `建模10张,渲染5张` |
| 客户与任务间使用全角连字符 | `甲方－建模×10` |
| 数量乘号使用英文字母 x/X | `建模x10张` |
| 任务描述为空，仅有数量 | `×5张` |
| 非标准计量单位 | `建模3幅` |
| 分隔符中英混用 | `任务A; 任务B；任务C` |

### Excel 列配置

配置 Excel 文件的解析规则：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 日期列 | 日期所在列的字母（如 B） | B |
| 人员列 | 各人员数据所在列，逗号分隔（如 C,D,E） | C,D |
| 表头行 | 表头所在行号，从 1 开始 | 1 |
| 人员列名 | 可选，覆盖列头作为人员名称，与人员列顺序一致 | 空 |

### 人员管理

- 手动添加人员到名册
- 删除人员
- 勾选"可显示"控制该人员是否出现在前台筛选下拉框中

### 自定义筛选项

每个筛选项包含：

| 字段 | 说明 |
|------|------|
| 显示名称 | 前台展示的标签名 |
| 关键词 | 从任务描述中匹配的字符串（包含即命中） |
| KPI 系数 | 数量 × 系数 = KPI 值 |
| 启用 | 控制是否在前台生效 |

支持增删改，保存后前台立即生效。

### Chart.js 资源地址

配置前台图表库的 CDN 地址，默认使用 staticfile.net 的国内镜像，可替换为私有 CDN 或本地地址。

### 前端模块顺序

5 个模块可拖拽排序，保存后前台按此顺序渲染各模块区域：

- 📈 KPI 总量趋势
- 📌 自定义筛选项 KPI 总览
- 👤 人员 KPI 对比
- 📅 每日/月度/人员图表
- 📋 明细表

### 前端配色方案

选择前台默认配色方案（共 5 套）。访客若从未手动切换，则加载后台在此设定的默认方案；已手动切换过的访客不受影响。

### 模块与图表标题

可自定义前台各位置的显示文字，留空则使用内置默认值：

| 配置项 | 默认值 |
|--------|--------|
| 页面标题 | PerformancePulse 绩效图表 |
| 页面副标题 | 按人员/时间维度分析 |
| KPI 总量趋势模块标题 | KPI 总量趋势 |
| 自定义筛选项模块标题 | 自定义筛选项 KPI 总览 |
| 人员对比模块标题 | 人员 KPI 对比 |
| 每日任务量图表标题 | 每日任务量 |
| 月度任务量图表标题 | 月度任务量 |
| 人员任务量图表标题 | 人员任务量 |
| 明细表模块标题 | 明细表 |

### 图表数据小数点位数

统一设置所有图表数值标签和卡片数值的小数保留位数（0~6），0 表示显示为整数。修改后保存，前台下次加载时生效。

## 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/performance` | 前台页面 |
| GET | `/admin` | 后台页面（需登录） |
| POST | `/admin/login` | 登录（JSON body：username/password） |
| POST | `/admin/logout` | 退出登录 |
| GET | `/admin/performance/config` | 获取完整配置（需登录） |
| PUT | `/admin/performance/config` | 保存配置（需登录） |
| POST | `/admin/performance/upload` | 上传 Excel 数据（需登录） |
| GET | `/admin/performance/records` | 获取所有记录（需登录） |
| GET | `/api/performance/summary` | 前台数据汇总（支持 person/from/to 参数） |
| POST | `/api/performance/export` | 导出 Excel（JSON body：person/from/to） |
| GET | `/api/app/chart-js` | 重定向到 Chart.js CDN 地址 |

### `/api/performance/summary` 返回结构

```json
{
  "totals": { "total_quantity", "total_kpi", "total_people", "total_days" },
  "by_day":    [{ "label": "2025-01-01", "value": 10 }, ...],
  "by_month":  [{ "label": "2025-01", "value": 120 }, ...],
  "by_year":   [{ "label": "2025", "value": 500 }, ...],
  "by_person": [{ "label": "张三", "value": 80 }, ...],
  "records":   [ ...工作日志明细, 含 kpi_factor / kpi_value 字段... ],
  "filter_totals":  [ ...每个自定义筛选项的汇总... ],
  "filter_series":  { "筛选项名": { "raw", "kpi", "clients_raw", "clients_kpi" } },
  "filter_clients": { "筛选项名": { "raw", "kpi" } },
  "config":    { "people", "people_enabled", "custom_filters", "app" }
}
```

## Excel 数据格式规范

每个单元格（对应一位员工某天的工作内容）支持多条任务，用**换行**或**分号（`;` / `；`）**分隔：

```
甲方A-建模*10张
甲方B-渲染×5张
材质调整3次
```

### 单条任务格式

```
[甲方名称-]任务描述[×/*/数量][单位]
```

| 部分 | 是否必须 | 示例 |
|------|----------|------|
| 甲方名称 | 可选，用半角 `-` 分隔 | `甲方A-` |
| 任务描述 | 必须 | `建模` |
| 数量乘号 | 可选，支持 `×`、`*` | `×` |
| 数量 | 可选，缺省为 1 | `10` |
| 单位 | 可选 | `张`、`个`、`次`、`套`、`小时` |

### 注意事项

- 多任务请用**换行或分号**分隔，不要用逗号
- 客户与任务之间使用**半角连字符** `-`，不要用全角 `－`
- 数量乘号使用 `×` 或 `*`，不要用英文字母 `x` / `X`
- 标准单位：张、个、次、套、小时
